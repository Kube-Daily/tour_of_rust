O que é utf-8

À medida que mais idiomas eram usados nos computadores, o mundo precisava representar mais caracteres de texto do que o ASCII permitia (1 byte permitia apenas 256 caracteres).

utf-8 foi introduzido com um comprimento variável de 1-4 bytes, o que aumentou muito o alcance de caracteres possíveis.

Uma vantagem de caracteres de taamnho variável é qe o texto não comumente tinha bytes desnecessários para ASCII muito comum (ainda apenas necessitando de 1 byte em utf-8).

Uma desvantagem de caracteres de tamanho variável é que o lookup de caracteres não mais pode ser feito rapidamente (tempo constante, O(1)) com acesso por índice simples (por exemplo, meu_texto[3], para obter o quarto caractere). É possível que os caracteres anteriores haviam tamanhos variáveis, alterando onde o quarto caracter de fato inicia na sequência de bytes.

No lugar, devemos iterar sobre uma sequência de bytes utf-8 para sabermos onde de fato uma sequência de caracteres utf-8 começa (tempo linear, O(n)).

Ferris: "Estou feliz de ter utf-8 para representar emojis dos meus amigos subaquáticos."
